## Lane 1: ship the skill + reference review gate

**Task:** turn the dropped-in SKILL.md/rationale.md (present in the
working tree but not yet in git — `main` holds only README.md, so this PR
adds both as new files) into an installable, process-complete skill repo
that dogfoods its own protocol: deterministic review gate, CI enforcement,
ruleset, and repo hygiene.

**Approach:** port the working gate from somnus `dev` (script, process
doc, CI job, tests), adapted: base `origin/main`, 4-leg battery
(claude-code-review, claude-security-review, codex-review,
codex-security), and a deterministically enforced model allowlist
(Claude: Opus 4.8 / Fable 5 or better; Codex: GPT 5.6 Sol or better —
"better" enters via a gated PR editing the allowlist). Fix SKILL.md's
dangling "review-gate companion" references to point at the in-repo
reference implementation. Write README (install: clone/symlink into
`~/.claude/skills/ship`), MIT LICENSE, changelog seed.

**Threat model impact:** introduces CI (GitHub Actions) and the gate
script as trust surface. Mitigations ported with the design: gate job runs
the base branch's copy of the script, `contents: read`, no persisted
credentials, stdlib only, no new dependencies beyond `pytest` (dev/CI
only). Changes to `.github/workflows/` or `scripts/review_gate.py` always
escalate to the human per the process doc.
